## About

The Security Assessments working group conducts collaborative security reviews of CNCF projects. Each assessment brings together TAG Security members, project maintainers, and open source community members to evaluate a project's security posture, identify gaps, and produce actionable recommendations.

This group's work has produced the manual [**Open and Secure: A Manual for Practicing Threat Modeling to Assess and Fortify Open Source Security**](https://github.com/cncf/tag-security/blob/main/community/assessments/Open_and_Secure.pdf), which provides a methodology for threat modeling open source projects.

## How Assessments Work

1. A CNCF project is selected for assessment
2. Community members volunteer to participate
3. The group conducts a structured review covering threat modeling, secure defaults, and vulnerability management
4. Findings and recommendations are published in the [assessments directory](https://github.com/cncf/tag-security/tree/main/community/assessments)

## Past Assessments

Assessment results are published on GitHub. See the [community/assessments](https://github.com/cncf/tag-security/tree/main/community/assessments) directory for completed reviews.
